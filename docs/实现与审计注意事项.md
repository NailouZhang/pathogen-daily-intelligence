# v1.3 实现与审计注意事项

## 1. 中文默认不等于允许模型补写

页面默认显示中文，但中文内容必须来自：

1. 原始中文来源；
2. 包内明确提供的 Demo 翻译；
3. 通过校验的大模型翻译。

如果三者均不存在，页面显示“中文翻译暂不可用”，不能退回到“用英文冒充中文”，也不能根据标题生成摘要。

## 2. 翻译调用范围

- 最终规范化后的全部文献和新闻文章均进入双语处理。
- `headline/brief/review` 且有证据的记录优先使用分析提示词，一次完成翻译和结构化分析。
- 其余记录进入批量翻译提示词。
- 同一输入由内容哈希缓存，后续重叠检索直接复用。

## 3. 科学排版与安全

外部标题属于不可信内容，不能直接关闭 HTML 转义。v1.3 使用白名单解析器，仅允许：

```text
sub, sup, i, em, strong, b, br
```

标签属性全部丢弃，`script`、`img`、`iframe`、事件处理器等不会执行。

翻译前科学片段会转换为：

```text
[[PDI_SCI_000]]
```

模型必须原样保留，缺失占位符的翻译会被拒绝。

## 4. 数字校验

- 中文标题必须保留英文标题中的全部数字。
- 完整中文摘要/摘录必须保留原文中的全部数字。
- 卡片摘要可以缩短内容，但不得添加新数字。
- `0` 仍只表示来源明确报告为零。

## 5. 新闻与事件

每篇 `NewsArticle` 均保存双语标题和摘录。日报静态主版面仍以 `PublicHealthEvent` 展示，防止多篇转载占据多个卡片；事件的双语内容来自其主来源文章。

Streamlit 另提供独立新闻文章页面，用于查看事件聚类前的每篇原文。

## 6. 邮件限制

邮件客户端不能可靠执行 JavaScript，因此邮件默认显示中文，并以小号文字保留英文原题，不提供交互切换按钮。

## 7. 模型失败

全部模型失败时：

- 原始中文内容仍正常显示；
- 已缓存翻译继续使用；
- 新英文内容显示中文翻译不可用；
- 英文按钮保留；
- 日报、JSON、Pages、Streamlit 和历史归档继续发布。

## 8. 生产后抽查

首次真实运行后重点抽查：

- 科学名称和蛋白名是否被错误翻译；
- 上下标和斜体是否完整；
- 数字、百分数和单位是否一致；
- 中文卡片摘要是否超出原文证据；
- 媒体称述是否被误写成官方确认；
- 相同记录是否命中缓存，而非每日重复调用模型。

## 9. 新闻正文与有限翻译文本

新闻分析最多使用 `content_policy.max_news_analysis_chars` 的正文证据；为了控制模型输出长度和翻译失败率，忠实中文摘录只翻译 `max_news_translation_chars` 范围内的源文本。结构化摘要、事件字段和来源链仍基于全部已编号正文证据。页面不会声称中文摘录等同于出版页面全文。

## 10. 多模型不是并行投票

- GitHub Models 首先处理批量翻译和单条分析。
- 只有未返回、结构错误或校验失败的条目才发送给 Gemini。
- 仍失败的条目才发送给 Groq。
- 已通过记录立即停止，不再调用其他模型。
- 每个对象保存 `attempt_chain`；调用级审计写入 `data/audit/llm_runs.jsonl`。

## 11. 提示词审查

所有生产提示词独立位于 `prompts/`。`prompt_review/production_prompts/` 是逐字审查副本；CI 会校验两处完全一致。`prompt_review/PRODUCTION_PROMPTS_COMBINED.md` 用于集中审阅，但运行时不读取合订本。


## v1.3.1：空值兼容与审计字段迁移

历史实体允许 `translation_audit` 为 `null`。Python 的 `dict.setdefault(key, {})` 在键已存在且值为 `null` 时不会替换该值，因此后续 `.update()` 会触发 `AttributeError`。本版统一通过 `ensure_dict_field()` 写入所有可变审计字段：

1. 字段已是对象时原样复用；
2. 字段为 `null`、列表、字符串或其他异常类型时，替换为空对象；
3. 保留后续模型尝试链和失败状态；
4. 所有模型不可用时仍完成 JSON、网页、邮件、RSS 和数据分支发布。

该迁移在内存中完成，不会修改原始来源事实字段。
