# 实现与审计注意事项

## 已实现的成熟首版边界

本版实现了从中英查询计划到日报发布的完整可运行链路，并提供 demo、CI、状态恢复、失败降级和可视化。它不把人工种子伪装成 ICTV 最终清单，也不承诺所有国家卫生机构已经完成注册。

## 关键事实边界

- 正式分类、物种状态和改名关系只能由 ICTV 等权威来源确认。
- DOI、PMID、卷期、病例数、日期、地点和宿主不得由模型猜测。
- 未知数字为 `null`；只有来源明确报告零时才能保存 `0`。
- 新闻发现器用于找到线索，不自动成为 A 级事实来源。
- 无摘要文献只显示元数据，不根据标题生成长摘要。

## 事件聚类

当前聚类使用病原、国家、事件类型、日期和标题相似度，并以较高阈值自动合并。该方案有意偏向 precision。生产成熟度仍需要人工标注的“应合并/不应合并/跨日更新”样本校准。

必须人工复核的高风险边界包括：

- 同城不同日期的独立病例；
- 人类病例与啮齿动物监测；
- 同省不同县事件；
- 媒体背景文章与当日通报；
- 相同病例数字但时间不同；
- 官方否认或数字更正。

## LLM 安全

网页正文属于不可信数据。提示词明确禁止把正文中的指令当成系统指令。模型结果经过 JSON、evidence ID、supporting item ID 和数字检查。验证失败时丢弃 AI 结果并保留原始摘要或规则结果。

当前数字校验较保守；复杂百分数、范围和格式化数字可能产生误拒绝，优先误拒绝而非误发布。

## API 与来源

- PubMed、Europe PMC、Crossref 和 Semantic Scholar 适配器均是独立发现源。
- Semantic Scholar 免费额度、GDELT 稳定性、Google News RSS 中转链接和 WHO 页面结构均可能变化。
- `source_health` 区分成功有结果、成功无结果、部分完成、失败和未启用。
- CDC/ECDC 在首版注册表中保留为未启用候选，待选择和测试稳定的病种页面、RSS、API 或 sitemap 后再启用。

## Git 存储

`intelligence-data` 适合中小规模 JSON/HTML 历史。随着实体和原始响应增长，应评估：

- 只在 Git 中保存最终实体和 issue；
- 原始响应仅作为短期 Actions artifact；
- 定期压缩 archive；
- 超过仓库体积阈值后迁移 SQLite、对象存储或数据库。

## 生产前检查

1. 设置真实联系邮箱和 User-Agent。
2. 审查 Profile、词典和来源注册表。
3. 以 demo 模式验证分支、Pages 和 Streamlit。
4. 以真实模式连续运行至少 14–28 天。
5. 人工抽查漏检、误合并、病例数字和来源等级。
6. 构建事件聚类金标准集。
7. 记录每次 Schema、规则、提示词和 Profile 版本变更。
