# v1.1 更新说明

## 发布架构变化

- 删除 `actions/upload-pages-artifact`；
- 删除 `actions/deploy-pages`；
- 删除 `deploy-pages` job；
- 删除 `pages: write` 和 `id-token: write`；
- 保留静态 HTML、RSS 和历史归档；
- Streamlit Community Cloud 成为唯一正式前端；
- `intelligence-data` 继续作为持久数据分支；
- Workflow 成功条件改为：生成、校验、artifact 和数据分支 push 成功。

## GitHub Actions 兼容性

- `actions/checkout@v6`；
- `actions/setup-python@v6`；
- `actions/upload-artifact@v7`。

这些版本面向当前 GitHub-hosted runner 的 Node.js 24 运行时，消除旧版 Node.js 20 deprecation 警告。

## Streamlit 增强

- 明确显示 GitHub 生产数据、本地缓存、Demo 或缺失状态；
- 显示 `intelligence-data` 分支和最近 commit；
- token 有 Actions: Read 时显示最近 Daily Workflow 状态；
- 手动清除缓存并刷新；
- GitHub 读取失败时回退到本地缓存；
- 本地缓存不存在时才回退 Demo；
- Demo 显示醒目标记，不能伪装成生产数据；
- 首页展示实体标题与事件摘要，不再只显示 ID；
- 增加“静态日报与下载”页面；
- 支持下载 DailyIssue JSON 和静态报纸 HTML。

## Secrets 收缩

Streamlit 只保存：

```toml
PDI_GITHUB_REPO = "NailouZhang/pathogen-daily-intelligence"
PDI_DATA_BRANCH = "intelligence-data"
GITHUB_DATA_TOKEN = ""
```

文献、LLM 和 SMTP Keys 只保存在 GitHub Actions Secrets。

## 保持不变的核心能力

- 中英文查询；
- 四个文献来源；
- 官方与新闻发现源；
- 五个核心数据对象；
- 文献实体去重；
- 新闻去重；
- 事件聚类和跨日更新；
- 规则过滤；
- 模型顺序降级；
- 无模型发布；
- `/tmp` 复制与 staged diff；
- 邮件、JSON、CSV、RSS 和历史审计。
