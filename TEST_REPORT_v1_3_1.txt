# Pathogen Daily Intelligence v1.3.1 测试与审计报告

## 版本目标

修复真实检索在翻译兜底阶段发生的：

```text
AttributeError: 'NoneType' object has no attribute 'update'
```

并确保模型凭据缺失、所有提供商失败、翻译校验失败或 LLM 编排层出现异常时，日报仍能以确定性模式完成发布。

## 根因确认

v1.3 新规范化对象使用：

```python
"translation_audit": None
```

最终翻译兜底使用：

```python
item.setdefault("translation_audit", {}).update(...)
```

当键存在且值为 `None` 时，`setdefault()` 返回 `None`，随后 `.update()` 崩溃。

## 修复范围

1. 新建文献和新闻对象默认使用 `translation_audit: {}`。
2. 新增 `ensure_dict_field()`，兼容历史 JSON 中的 `null` 和异常类型。
3. 加固 `translation_audit`、`processing_audit`、`retrieval_audit`、`display_summary`、`content`、`abstract`、`quality` 和 `fingerprints` 的写入。
4. 翻译全部提供商失败后写入：
   - `validation_status=translation_unavailable_after_all_providers`
   - 完整 `attempt_chain`
   - `processing_audit.translation`
5. 新增 LLM 编排级最终异常边界：即使 LLM 子系统发生未预期 Python 异常，也继续生成 JSON、HTML、邮件、RSS 和数据分支内容。
6. CI 增加无任何模型凭据、但启用 LLM 路径的命令级烟雾测试。
7. 生产提示词和模型路由未改变，提示词审查副本继续逐字节一致检查。
